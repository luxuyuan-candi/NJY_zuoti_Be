# 微信小程序页面功能与跳转关系设计文档

> 当前版本基于“考试类 + 课程类”的移动端刷题小程序场景设计，覆盖小程序页面、后台管理端页面、核心功能、布局关系和跳转流程。

## 1. 项目定位

本小程序面向需要在移动端进行题目练习、模拟考试、错题复盘和学习进度管理的用户。核心目标是降低刷题入口成本，让用户可以快速开始练习，并持续追踪学习效果。

## 2. 设计假设

- 用户不需要手动登录；小程序通过 `wx.login` 获取 code，后端换取 `openid`，并以 `openid` 作为用户身份主键。
- 用户可在“身份配置”页补充头像、昵称、邮箱；头像通过微信头像接口选择。
- “我的”页顶部身份模块不是登录按钮，而是资料入口；无头像时只显示空白头像圆圈，不展示默认头像。
- “我的”页顶部身份模块展示头像、昵称和角色；昵称为空时展示“匿名”。
- 用户角色分为“游客”、“普通用户”、“管理员”、“超级管理员”。
- “管理员”和“超级管理员”可以通过工具区“人员管理”配置用户角色；只有“超级管理员”可以配置“管理员”权限。
- 身份资料保存后端落库，头像文件保存到 MinIO，页面不直接持有 MinIO 内网地址。
- 题库业务范围包含考试类题库和课程类题库。
- 题库按“业务类型 / 科目或课程 / 分类 / 章节 / 套卷”组织。
- 不提供付费题库、会员、兑换码或订单能力。
- 只有已注册且已由管理员授权的用户可以查看题库并进行练习。
- 做题模式包含章节练习、随机练习、专项练习、模拟考试、错题重做、收藏题练习。
- 题型仅支持客观题，包含单选、多选、判断；不支持主观题，不涉及人工批改或 AI 批改。
- 用户需要查看练习记录、错题、收藏、成绩趋势。
- 错题支持“多次答对后自动移出”和“用户手动移出”两种策略。
- 小程序需要支持题目离线缓存。
- 后台管理端负责用户授权、题库维护、内容推广、奖牌排名和反馈处理。

## 3. 信息架构

小程序采用底部 Tab 作为一级导航，建议包含 4 个主入口：

| Tab | 页面 | 主要职责 |
| --- | --- | --- |
| 首页 | `pages/home/index` | 教学视频展示、学习推广内容 |
| 题库 | `pages/bank/index` | 考试类 / 课程类题库、分类、章节、套卷选择 |
| 记录 | `pages/record/index` | 错题本、题目收藏、考试成绩、趋势统计 |
| 我的 | `pages/profile/index` | 身份资料、奖牌排名、工具区 |

## 4. 页面清单

### 4.1 首页

路径：`pages/home/index`

功能：

- 顶部展示教学视频，作为首页首屏核心内容。
- 展示学习推广内容，例如课程介绍、学习方法、活动海报、备考提醒等。
- 支持推广内容跳转到详情页、外部文章或指定题库。
- 首页不作为主要做题入口，练习入口主要放在题库页和记录页中。

布局：

- 顶部：教学视频播放器或视频封面，支持播放、暂停、全屏。
- 中部：学习推广内容列表，可采用 Banner、图文卡片或活动入口。
- 下方：更多推广内容或公告信息。

跳转：

- 点击教学视频 -> 视频播放页或当前页内播放。
- 点击推广内容 -> 推广详情页 / 文章详情页 / 指定题库页。
- 点击公告 -> 公告详情页。

### 4.2 题库页

路径：`pages/bank/index`

功能：

- 展示当前用户可访问的 MongoDB `practice_sets` 习题集。
- 第一阶段直接展示三个理论题库：初级、中级、高级。
- 支持按等级切换：全部、初级、中级、高级。
- 支持按题库名称和等级搜索。
- 展示每个题库的题目总数、等级标签和来源描述。
- 未授权用户进入题库页时展示授权提示，不展示题库内容。

布局：

- 顶部：搜索框。
- 中部：横向分类筛选。
- 下方：题库列表，每项展示名称、描述、题量、等级。

跳转：

- 点击题库 -> 章节列表页。
- 点击搜索结果 -> 章节列表页或套卷详情页。

### 4.3 章节列表页

路径：`pages/chapter/index`

功能：

- 展示指定题库下按 `knowledge.pathNames` 聚合后的章节分组。
- 当前章节口径为“知识点叶子节点之前的层级路径”。
- 展示章节题量和所属上级路径。
- 提供章节练习入口和整套题库综合练习入口。
- 提供“错题重做”入口，直接进入基于当前用户错题本的练习设置。

布局：

- 顶部：题库名称、等级、总题量。
- 下方：章节列表。
- 底部：固定按钮“错题重做”和“开始综合练习”。

跳转：

- 点击章节 -> 练习设置页。
- 点击“开始综合练习” -> 练习设置页。
- 点击“错题重做” -> 错题重做练习设置页。

### 4.4 练习设置页

路径：`pages/practice/settings`

功能：

- 设置做题范围、题目数量、出题顺序。
- 设置是否显示答案解析。
- 第一阶段题目数量通过滚轮选择器从 `1` 到当前最大题目数。
- 第一阶段默认采用随机出题。
- 第一阶段不提供“只做未做题”筛选。
- 显示当前章节或整套题库总题数，并按可用题量动态生成可选题目数。

布局：

- 顶部：练习对象名称。
- 中部：题量滚轮选择器、随机出题标识、解析策略开关。
- 底部：主按钮“开始练习”。

跳转：

- 点击“开始练习” -> 做题页。
- 返回 -> 章节列表页 / 题库页 / 错题列表页。

### 4.5 做题页

路径：`pages/practice/answer`

功能：

- 展示题干、选项、题号、当前进度。
- 支持单选、多选、判断等题型作答。
- 支持上一题、下一题、提交答案。
- 支持标记疑问、查看解析。
- 已答题允许重新选择并再次提交，结果以最后一次提交为准。
- 支持错题自动记录。
- 支持考试模式下隐藏解析，并在交卷后统一展示结果。

布局：

- 顶部：进度条、题号、倒计时或练习耗时。
- 中部：题干与选项区域。
- 下方：解析区，练习模式可答后展示，考试模式不展示。
- “答后显示解析”关闭时：
  - 提交后不自动显示正确选项
  - 提交后展示“查看解析”按钮
  - 用户点击“查看解析”后才显示正确选项和解析区
- 底部：上一题、下一题、提交 / 完成按钮。
- 悬浮：题目列表入口、收藏按钮。

跳转：

- 点击题目列表 -> 题目导航弹层。
- 点击完成练习 -> 练习结果页。
- 考试模式点击交卷 -> 考试结果页。
- 点击返回且有未保存进度 -> 二次确认弹窗。

### 4.6 题目导航弹层

路径：做题页内弹层组件，不建议单独成页。

功能：

- 展示全部题号。
- 区分已答、未答、当前题、标记疑问状态。
- 支持快速跳转到指定题目。

布局：

- 顶部：状态图例。
- 中部：题号网格。
- 底部：关闭按钮。

状态颜色规则：

- 当前题：蓝色
- 已做题：绿色
- 存疑题：黄色
- 只有全部题目都已提交后，底部主按钮才显示“完成”；如果当前位于最后一题但仍有未答题，点击“下一题”应跳转到第一道未答题。

### 4.7 练习结果页

路径：`pages/practice/result`

功能：

- 展示本次练习总题数、正确数、错误数、正确率、耗时。
- 展示按题型或章节拆分的表现。
- 提供查看错题、再练一次、返回题库入口。
- “查看错题”只展示本次完成记录中的错题，不进入全量错题本。
- 本次错题和错题本中的每一条错题都支持进入详情页，查看题干、选项、用户选择、正确答案和答案解析。

布局：

- 顶部：结果摘要。
- 中部：统计指标和薄弱点提示。
- 下方：题目明细列表。
- 底部：操作按钮。

跳转：

- 点击“查看错题” -> 错题列表页，筛选本次练习错题。
- 点击“再练一次” -> 使用 `reLaunch` 直接回到题库页。
- 点击“返回题库” -> 章节列表页。
- 练习完成后使用页面替换进入结果页，结果页返回时不再回到刚刚的做题页。

### 4.8 套卷列表页

路径：`pages/paper/index`

功能：

- 展示模拟卷、真题卷、章节卷。
- 展示题量、建议用时、已完成次数、最高分。
- 支持按年份、类型、难度筛选。

布局：

- 顶部：筛选栏。
- 下方：套卷列表。

跳转：

- 点击套卷 -> 考试说明页。

### 4.9 考试说明页

路径：`pages/exam/intro`

功能：

- 展示考试名称、题量、满分、限时、规则。
- 提醒考试中不可查看解析。
- 提供开始考试按钮。

布局：

- 顶部：考试标题。
- 中部：考试规则和计分方式。
- 底部：开始考试按钮。

跳转：

- 点击“开始考试” -> 做题页，进入考试模式。
- 返回 -> 套卷列表页。

### 4.10 考试结果页

路径：`pages/exam/result`

功能：

- 展示分数、正确率、排名或超过人数。
- 展示各题型得分情况。
- 支持查看解析、重做本卷、返回套卷列表。

布局：

- 顶部：分数和评价。
- 中部：答题统计。
- 下方：题目明细。
- 底部：操作按钮。

跳转：

- 点击“查看解析” -> 做题页，只读解析模式。
- 点击“重做本卷” -> 考试说明页或做题页。
- 点击“返回套卷” -> 套卷列表页。

### 4.11 错题列表页

路径：`pages/mistake/index`

功能：

- 展示用户错题集合。
- 支持按题库、章节、题型、时间筛选。
- 支持移出错题本。
- 支持错题重做。
- 点击“开始错题重做”后，练习设置页题量来自当前错题本实际题数，而不是题库章节总数。
- 错题本聚合用户所有错题，按题目展示，并显示累计做错次数 `wrongTimes`。
- 同一道题多次做错时，做错次数会持续累计。
- 支持用户手动移出错题本；后续再次做错时重新进入错题本。
- 点击任意错题进入详情页后，可查看题干、选项、正确答案和答案解析；如果入口来自“本次错题”，额外展示该次作答的用户选择。

布局：

- 顶部：筛选栏。
- 中部：错题统计摘要。
- 下方：错题列表。
- 底部：开始重做按钮。

跳转：

- 点击错题 -> 错题详情页或做题页只读模式。
- 点击“开始重做” -> 练习设置页或做题页。

### 4.12 收藏题列表页

路径：`pages/favorite/index`

功能：

- 展示用户收藏题。
- 支持按题库、章节、题型筛选。
- 支持取消收藏。
- 支持收藏题专项练习。

布局：

- 顶部：筛选栏。
- 下方：收藏题列表。
- 底部：开始练习按钮。

跳转：

- 点击题目 -> 题目详情页或做题页只读模式。
- 点击“开始练习” -> 做题页。

### 4.13 记录页

路径：`pages/record/index`

功能：

- 只统计用户点击“完成”后的已完成练习或考试结果；做题过程中选择答案、提交单题答案不进入记录统计。
- 正确率按总作答次数统计，不按题目去重；同一道题多次做错时，多次错误都会分别计入统计。
- “记录”页相关统计和最近记录均来自后端持久化数据，而不是本地缓存。
- 提供错题本入口，展示错题数量和最近新增错题。
- 提供题目收藏入口，展示收藏题数量。
- 展示考试成绩，支持查看历史考试结果。
- 展示趋势统计，例如正确率趋势、考试成绩趋势、做题数量趋势。
- 未产生已完成记录前，总做题数、正确率、考试数显示为 `0`，错题本和趋势统计入口点击后提示先完成一次练习。
- 当前“考试数”展示口径按“完成次数”统计，即每次练习或考试点击“完成”后计 `1` 次。

布局：

- 顶部：学习数据概览，展示总做题数、正确率、考试次数等核心指标。
- 中部：错题本、题目收藏、趋势统计三个快捷入口。
- 下方：最近完成记录列表，时间按东八区显示并精确到分钟。

跳转：

- 点击“错题本” -> 错题列表页。
- 点击“题目收藏” -> 收藏题列表页。
- 点击趋势统计 -> 趋势统计详情页。
- 点击最近记录 -> 练习结果页或考试结果页。

### 4.14 我的页

路径：`pages/profile/index`

功能：

- 自动获取并建立当前用户身份。
- 展示头像、昵称、角色等基础信息；昵称未配置时展示“匿名”，未配置头像时顶部圆圈保持空白。
- 点击整个顶部身份模块进入身份配置页。
- 展示用户已获取的奖牌、勋章或成就。
- 展示用户排名情况，包括总榜排名、周榜排名和个人当前排名。
- 提供工具区，承载反馈、人员管理等工具；人员管理只对管理员和超级管理员展示。

布局：

- 顶部：用户身份卡片，左侧圆形头像，未配置头像显示空白圆圈；中部展示昵称或“匿名”、角色标签。
- 中部：奖牌与排名区域。
- 下方：工具区，采用宫格或列表形式，先预留反馈等入口。

跳转：

- 点击身份模块 -> 身份配置页。
- 点击奖牌 -> 奖牌详情页。
- 点击排名 -> 排名页。
- 点击反馈 -> 反馈页。
- 管理员或超级管理员点击人员管理 -> 人员管理页。

### 4.15 身份配置页

路径：`pages/auth/login`

功能：

- 配置用户头像、昵称、邮箱。
- 头像通过微信头像接口选择，使用单个“头像选择”按钮触发。
- 保存后，昵称和邮箱写入 MySQL，头像文件写入 MinIO，头像 URL 保存到用户资料。
- 页面路径沿用 `pages/auth/login`，但页面语义从“登录授权”调整为“身份配置”。

布局：

- 顶部：身份配置说明。
- 中部：圆形头像预览、头像选择按钮。
- 下方：昵称输入框、邮箱输入框、保存按钮。

交互规则：

- 进入页面时先调用身份初始化逻辑，确保本地存在 token 和当前用户资料。
- 未选择头像时头像预览保持空白圆形边框。
- 点击“头像选择”直接调用微信头像接口，拿到头像临时文件后进入保存流程。
- 前端将头像转为 base64 提交给后端；后端返回 MinIO 公开 URL 后更新本地用户资料。
- 邮箱允许为空；填写时前端做基础格式校验，后端仍必须二次校验。
- 保存中禁用保存按钮，避免重复提交。
- 保存失败时保留当前表单和头像预览，用户可重试或更换头像。

跳转：

- 保存成功 -> 返回来源页。

## 6. 关键业务流程

### 6.1 首次进入小程序

1. 用户进入首页。
2. 小程序启动时立即调用 `wx.login`。
3. 后端通过微信 code 换取 `openid`，以 `openid` 创建或读取用户记录，并返回 token。
4. 新用户默认角色为“游客”。
5. 前端将 token 和用户资料缓存到本地，后续接口通过 `Authorization: Bearer <token>` 访问。
6. 首页展示教学视频和学习推广内容，公开内容可直接浏览。
7. 如果启动阶段拿着旧 token 拉取用户资料失败，前端应清理本地身份缓存并重新执行 `wx.login`，不能继续回退展示本地旧用户信息。

### 6.2 章节练习流程

1. 用户从题库页进入题库，也可以从首页推广内容跳转到指定题库。
2. 选择章节。
3. 进入练习设置页。
4. 设置题量、顺序、解析策略。
5. 开始做题。
6. 完成后进入练习结果页。
7. 用户可查看错题、再练一次或返回章节。

### 6.3 模拟考试流程

1. 用户从题库页进入套卷列表，也可以从首页推广内容跳转到指定套卷。
2. 选择套卷。
3. 阅读考试说明。
4. 开始考试。
5. 考试中显示倒计时，不展示答案解析。
6. 用户主动交卷或倒计时结束自动交卷。
7. 进入考试结果页。
8. 用户可查看解析或重做本卷。

### 6.4 错题复盘流程

1. 用户从记录页进入错题列表。
2. 筛选题库、章节或题型。
3. 开始错题重做。
4. 答对后可根据策略自动移出错题本，或保留历史错题记录。
5. 完成后进入练习结果页。

### 6.5 收藏题练习流程

1. 用户在做题页点击收藏。
2. 收藏题进入收藏题列表。
3. 用户从记录页进入收藏题。
4. 选择题目查看，或批量开始收藏题练习。

### 6.6 首页内容浏览流程

1. 用户进入首页。
2. 顶部查看教学视频，可直接播放或进入视频播放页。
3. 向下浏览学习推广内容。
4. 点击推广内容后进入详情页，详情页可按运营配置跳转到题库、套卷、活动或外部文章。

### 6.7 我的页个人信息流程

1. 用户进入我的页。
2. 页面自动确保身份存在，并展示头像、昵称、角色、奖牌和排名信息。
3. 未配置头像时，顶部头像圆圈显示为空白。
4. 点击整个身份模块进入身份配置页。
5. 在身份配置页选择头像来源，填写昵称和邮箱。
6. 点击保存后，头像写入 MinIO，昵称、邮箱和头像 URL 写入 MySQL。
7. 返回我的页后展示最新头像和资料。
8. 点击奖牌进入奖牌详情页，点击排名进入排名页。
9. 点击工具区入口进入反馈或人员管理等工具页面。

### 6.8 身份资料保存状态流

```mermaid
stateDiagram-v2
  [*] --> EnsureIdentity: 进入我的页或身份配置页
  EnsureIdentity --> ProfileReady: 已获取 token 和用户资料
  EnsureIdentity --> IdentityError: wx.login 或后端换取 openid 失败
  ProfileReady --> Editing: 点击身份模块
  Editing --> AvatarSelected: 选择微信头像
  Editing --> Saving: 只修改昵称或邮箱
  AvatarSelected --> Saving: 提交头像 base64 和资料
  Saving --> Saved: 后端保存 MySQL 和 MinIO 成功
  Saving --> SaveError: 上传或落库失败
  SaveError --> Editing: 保留表单重试
  Saved --> ProfileReady: 刷新我的页资料
```

状态说明：

| 状态 | 页面要求 |
| --- | --- |
| `EnsureIdentity` | 可以显示加载态，但不应要求用户点击登录 |
| `ProfileReady` | 我的页展示当前资料；头像为空时展示空白圆圈 |
| `Editing` | 身份配置页允许修改昵称、邮箱和头像 |
| `Saving` | 禁用保存按钮，防止重复提交 |
| `SaveError` | 保留用户已填写内容，不清空头像预览 |
| `Saved` | 更新本地缓存后返回来源页或刷新我的页 |

## 7. 页面状态设计

### 7.1 身份与授权状态

| 状态 | 页面表现 |
| --- | --- |
| 未建立身份 | 小程序启动后立即获取 openid；如失败则允许公开内容页降级展示 |
| 已建立身份未配置资料 | 我的页头像为空白圆圈，昵称显示“匿名” |
| 已配置资料未授权 | 我的页显示用户资料；题库页提示等待管理员授权，不展示题库 |
| 已配置资料已授权 | 题库页展示授权题库；记录页显示错题、收藏、成绩、趋势；我的页显示用户资料、奖牌和排名 |
| 身份 token 失效 | 清空本地 token 和 user，自动重新调用 `wx.login` 建立身份，保留当前目标页面 |

### 7.2 数据空状态

| 场景 | 空状态文案 | 操作 |
| --- | --- | --- |
| 无错题 | 暂无错题，继续保持 | 去练习 |
| 无收藏 | 暂无收藏题 | 去题库 |
| 无记录 | 暂无练习记录 | 开始练习 |
| 无奖牌 | 暂未获得奖牌 | 去练习 |
| 无排名 | 暂无排名数据 | 稍后查看 |
| 无推广内容 | 暂无学习内容 | 稍后重试 |
| 无题库 | 暂无可用题库 | 联系管理员或稍后重试 |
| 未授权题库 | 暂未开通题库权限 | 联系管理员 |

### 7.3 异常状态

| 场景 | 处理方式 |
| --- | --- |
| 网络异常 | 展示重试按钮，保留页面上下文 |
| 题目加载失败 | 提示重新加载，不清空已答状态 |
| 交卷失败 | 本地临时保存答题数据，允许重试提交 |
| 身份获取失败 | 停留当前页，提示稍后重试 |
| 资料保存失败 | 保留表单内容，提示稍后重试 |
| 头像上传失败 | 保留本地预览，提示重新保存或更换头像 |
| 身份初始化拿到旧缓存 | 不允许直接回退展示旧 user 缓存，必须重新登录拉取服务端资料 |
| 授权失效 | 清空受限题库入口和离线缓存，提示重新联系管理员 |
| 离线缓存失败 | 展示失败原因，允许重新缓存 |

### 7.4 资料展示优先级

我的页顶部资料按以下优先级展示：

| 字段 | 展示规则 |
| --- | --- |
| 头像 | 有 `avatarUrl` 时展示图片；无头像时展示空白圆形占位 |
| 昵称 | 有 `nickname` 时展示昵称；无昵称时展示“匿名” |
| 邮箱 | 有邮箱时在身份配置页回填；我的页可按空间决定是否展示 |
| 角色 | 展示“游客”、“普通用户”、“管理员”或“超级管理员” |
| 授权状态 | 题库页使用授权状态决定是否展示题库，我的页可展示当前状态提示 |

## 8. 权限与缓存规则

### 8.1 题库访问权限

- 首页教学视频和学习推广内容默认可浏览，具体内容是否需要登录可由后台配置。
- 题库页、章节列表页、做题页、错题本、收藏题和考试成绩需要用户身份 token。
- 用户建立 openid 身份后仍需管理员授权，才可以查看被授权的考试类或课程类题库。
- 授权范围可以精确到业务类型、题库、课程、章节或套卷，第一阶段建议精确到题库或课程。
- 后端接口必须按用户授权范围过滤题库数据，前端只负责展示授权结果。

### 8.2 离线缓存规则

- 用户只能缓存自己已被授权访问的题目。
- 缓存内容包含题干、选项、正确答案、解析、题型、章节、套卷归属和版本号。
- 缓存不包含用户无权限访问的题库，也不缓存后台已下架或已撤权的题库。
- 每个题库或章节需要维护缓存版本号；版本变化时提示用户更新缓存。
- 用户授权失效后，小程序应清理对应题库的离线缓存。
- 离线作答产生的记录需要本地暂存，网络恢复后同步到后端。

## 9. 做题页交互规则

- 单选题：点击选项后立即记录答案，可根据练习设置决定是否立即显示解析。
- 多选题：点击选项切换选中状态，点击提交后判题。
- 判断题：点击“正确 / 错误”后记录答案。
- 考试模式：答题过程中不展示正确答案和解析。
- 练习模式：可配置“答后显示解析”或“完成后统一查看解析”。
- 收藏：再次点击取消收藏。
- 标记疑问：只影响用户个人标记，不影响判题。
- 错题：同一道错题连续或累计多次答对后自动移出，具体次数由后台配置；用户也可以在错题列表手动移出。
- 退出：存在未完成作答时弹窗确认，避免误退出。

## 13. 后台管理端页面设计

后台管理端建议采用 Web 管理系统，主要服务管理员、运营人员和题库维护人员。第一阶段重点是保证“用户授权 -> 题库维护 -> 内容配置 -> 数据查看”的基础链路闭环。

### 13.1 后台登录页

路径建议：`/admin/login`

功能：

- 管理员账号登录。
- 支持退出登录和登录态过期处理。
- 可预留角色权限控制。

跳转：

- 登录成功 -> 后台首页。

### 13.2 后台首页

路径建议：`/admin/dashboard`

功能：

- 展示题库数量、注册用户数、已授权用户数、今日做题数、考试次数等核心指标。
- 展示待处理反馈、最近题库更新、最近用户授权记录。

布局：

- 顶部：核心数据卡片。
- 中部：趋势图和待办事项。
- 下方：最近操作记录。

### 13.3 用户管理与授权页

路径建议：`/admin/users`

功能：

- 查看注册用户列表。
- 按 openid、昵称、邮箱、注册时间、授权状态筛选用户。
- 给用户授权考试类或课程类题库。
- 取消用户题库授权。
- 查看用户做题记录、考试成绩、错题数量、收藏数量。

布局：

- 顶部：筛选区。
- 中部：用户表格。
- 右侧或弹窗：授权配置面板。

跳转：

- 点击用户 -> 用户详情页。
- 点击授权 -> 授权弹窗。

### 13.3.1 小程序人员管理页

路径：`pages/admin/users/index`

功能：

- 仅管理员和超级管理员可从“我的”页工具区进入。
- 展示用户头像、昵称、当前角色。

- 管理员可将其他普通用户或游客设置为普通用户或游客。
- 不允许用户修改自己的角色。
- 小程序端不开放授予超级管理员权限，超级管理员应由数据库初始化或运维后台维护。

跳转：

- 我的页工具区“人员管理” -> 人员管理页。
- 配置角色成功 -> 刷新人员列表。
- 无权限访问 -> 提示后返回上一页。

### 13.4 题库管理页

- 可记录处理备注。

第一阶段同步实现的关键能力：

- 管理员授权后用户才可查看题库。
- 用户身份自动以 openid 建立，不需要手动登录。
- 我的页身份资料可保存头像、昵称和邮箱，头像存储到 MinIO。
- 客观题做题、判题、解析展示。
- 错题本和题目收藏。
- 考试成绩记录。
- 总榜、周榜和个人当前排名的基础展示。
- 题目离线缓存和缓存版本更新提示。

暂缓实现：

- 复杂奖牌规则
- 班级管理
- 复杂趋势图
- 高级数据导出
- 多角色细粒度权限

这样可以先闭环“首页内容展示 -> 授权用户选题 -> 做题或考试 -> 结果 -> 记录复盘 -> 我的个人信息 -> 后台维护题库与授权”的主路径，再逐步扩展复杂奖牌、运营活动和更多工具能力。
